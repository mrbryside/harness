# Tool / Agent Guide: Code Diff Rendering

When an AI agent edits a file, it tells the TUI two things: the **old text** and the **new text**, plus **where the new text starts** (`StartLine`).  The renderer then shows a real unified diff with correct line numbers and syntax highlighting.

`EndLine` is computed automatically from `NewContent` — the agent does not provide it.

---

## What the Agent Must Provide

```go
type CodeDiff struct {
    Path       string // file path
    OldContent string // exact text currently in the file
    NewContent string // replacement text
    StartLine  int    // line number of the FIRST new line
}
```

- **Path** — the file being edited.
- **OldContent** — read from disk before editing (must match exactly).
- **NewContent** — generated by the agent.
- **StartLine** — line number where the replacement begins.

`EndLine` is derived internally: `StartLine + strings.Count(NewContent, "\n")`.

---

## How EndLine Is Computed (Internal)

The framework counts newlines in `NewContent` and adds them to `StartLine`:

```go
endLine := startLine + strings.Count(newContent, "\n")
```

| NewContent | `\n` count | lineCount | If StartLine=5 |
|---|---|---|---|
| `return 42` | 0 | 1 | EndLine = **5** |
| `if x {\n  return 42\n}` | 2 | 3 | EndLine = **7** |
| 8 lines of code | 7 | 8 | EndLine = **12** |

The agent never thinks about `EndLine`.

## Example 1: Single-line replacement

**File before edit (`demos/helper.go`):**

```go
10: func greet(name string) string {
11:     return "Hello, " + name
12: }
```

**Agent:** *Replace line 11.*

```go
chat.AppendCodeDiff(components.CodeDiff{
    Path:       "demos/helper.go",
    OldContent: `    return "Hello, " + name`,
    NewContent: `    return fmt.Sprintf("Hello, %s", name)`,
    StartLine:  11,
})
```

**Framework computes:** `EndLine = 11 + 0 = 11` (single line, no newlines).

## Example 2: Multi-line replacement (growing)

**File before edit (`demos/agent_demo.go`):**

```go
24: func (c *agentCommand) Execute(args string) Result {
25:     return Result{
26:         Chat:  "Available agents: coder, reviewer, architect",
27:         Toast: "✓ Agent list",
28:     }
29: }
```

**Agent:** *Replace lines 25-28 with a bigger block starting at line 25.*

```go
chat.AppendCodeDiff(components.CodeDiff{
    Path: "demos/agent_demo.go",
    OldContent: `    return Result{
        Chat:  "Available agents: coder, reviewer, architect",
        Toast: "✓ Agent list",
    }`,
    NewContent: `    // Validate agent name
    if args == "" {
        return Result{Chat: "Error", Toast: "Missing"}
    }
    return Result{
        Chat:  fmt.Sprintf("Switched to %s", args),
        Toast: "✓ Done",
    }`,
    StartLine: 25,
})
```

**Framework computes:** `EndLine = 25 + 7 = 32` (8 lines, 7 newlines).

**Rendered output:**


## Example 3: Shrinking replacement

**File before edit (`demos/config.go`):**

```go
40: func loadConfig() Config {
41:     return Config{
42:         Host: "localhost",
43:         Port: 8080,
44:         Debug: false,
45:     }
46: }
```

**Agent:** *Collapse lines 41-45 to one line.*

```go
chat.AppendCodeDiff(components.CodeDiff{
    Path: "demos/config.go",
    OldContent: `    return Config{
        Host: "localhost",
        Port: 8080,
        Debug: false,
    }`,
    NewContent: `    return Config{Host: "localhost", Port: 8080, Debug: false}`,
    StartLine:  41,
})
```

**Framework computes:** `EndLine = 41 + 0 = 41` (single line).

---

## Summary

1. **Read** old text from disk exactly.
2. **Generate** new text.
3. **Know** which line the replacement starts at (`StartLine`).
4. **Call** `AppendCodeDiff` with four fields.

The framework handles `EndLine` automatically.
