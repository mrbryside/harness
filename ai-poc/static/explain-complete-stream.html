Complete Stream Assembly Explained

All parts need assembly: Content, Reasoning, and Tool Calls

Key Question
"Do we only assemble tool calls?"
No! Everything must be assembled!

What needs assembly in a stream?

In choices[0].delta, you may see these three types mixed together:

Field Assembly Method Description
content
Response text 		content += delta.content Simple string concatenation (like normal chat)
reasoning_content
Thinking 		reasoning += delta.reasoning_content Same as content but internal thoughts (always before content)
tool_calls
Tool calls 		map[index] += arguments Most complex! Use map by index and append arguments (see other page)
🔄 Full Flow: How to Assemble Response?
📤 Step 1: Create Empty Assistant Message
Create empty structure to receive stream data
assistantMessage = { role: "assistant", content: "", // waiting for content chunks reasoning_content: "", // waiting for reasoning chunks tool_calls: [] // waiting for tool call chunks } toolCallBuffer = {} // map to store tool calls during stream
⬇
📥 Step 2: Read Each Chunk from Stream
Each chunk comes as SSE (Server-Sent Events)
// Example chunk 1 data: { "choices": [{ "index": 0, "delta": { "role": "assistant", // only comes once "reasoning_content": "Let" // starts thinking }, "finish_reason": null }] }
⬇
🔨 Step 3: Assemble Each Part
Check what's in delta and append accordingly
for each chunk in stream: delta = chunk.choices[0].delta // 3.1 Assemble CONTENT (if present) if delta.content exists: assistantMessage.content += delta.content // "H" + "ello" + "!" → "Hello!" // 3.2 Assemble REASONING (if present) if delta.reasoning_content exists: assistantMessage.reasoning_content += delta.reasoning_content // "Let" + " me" + " think" → "Let me think" // 3.3 Assemble TOOL CALLS (if present - most complex) if delta.tool_calls exists: for each tc in delta.tool_calls: idx = tc.index if toolCallBuffer[idx] not exists: toolCallBuffer[idx] = { id, type, function: { name, arguments: "" } } toolCallBuffer[idx].function.arguments += tc.function.arguments // Append arguments piece by piece!
⬇
✅ Step 4: Stream Ends: Convert to Complete Message
When finish_reason is not null, stream is done
// Convert toolCallBuffer to array for i from 0 to len(toolCallBuffer) - 1: assistantMessage.tool_calls.push(toolCallBuffer[i]) // Now assistantMessage has complete data: { role: "assistant", content: "Hello! How can I help?", reasoning_content: "Let me think...", tool_calls: [ { id: "call_abc", function: { name: "bash", arguments: "{...}" } }] }
🎭 3 Scenarios: What Streams Come?

Scenario: AI responds with normal text (no reasoning, no tool)

User: "Hello!"

What streams: Only delta.content

1
content
{ "delta": { "role": "assistant", "content": "Hello" } }

content = "Hello"

2
content
{ "delta": { "content": "!" } }

content = "Hello!" (appended)

3
finish
{ "delta": {}, "finish_reason": "stop" }

Stream done! Got "Hello!" complete

✅ Final Result (content only)
{ role: "assistant", content: "Hello!", reasoning_content: "", // empty tool_calls: [] // empty }

Scenario: AI thinks before responding (Reasoning Model like o1, QwQ)

User: "What is 15 × 24?"

What streams: delta.reasoning_content first, then delta.content

Order: Reasoning (multiple chunks) → Content (multiple chunks) → Finish

1
reasoning_content
{ "delta": { "role": "assistant", "reasoning_content": "Let" } }

reasoning = "Let"

2
reasoning_content
{ "delta": { "reasoning_content": " me calculate" } }

reasoning = "Let me calculate"

3
reasoning_content
{ "delta": { "reasoning_content": "... 15 x 20 = 300, 15 x 4 = 60..." } }

reasoning = "Let me calculate... 15 x 20 = 300, 15 x 4 = 60..."

4
content (starts after reasoning)
{ "delta": { "content": "15" } }

content = "15" (reasoning stopped, now content)

5
content
{ "delta": { "content": " x 24 = 360" } }

content = "15 x 24 = 360"

6
finish
{ "delta": {}, "finish_reason": "stop" }

Stream done!

✅ Final Result (both reasoning + content)
{ role: "assistant", content: "15 x 24 = 360", // what user sees reasoning_content: "Let me calculate... 15 x 20 = 300, 15 x 4 = 60...", // AI's thoughts (hidden) tool_calls: [] // empty (no tools) }

Notice: reasoning_content comes before content! They don't come together.

Scenario: AI uses Tool (as explained in explain-tools-stream.html)

User: "List files"

What streams: delta.tool_calls in multiple chunks (built piece by piece)

No content! Because AI doesn't text back, but calls tool instead.

1
tool_calls (init)
{ "delta": { "tool_calls": [{ "index": 0, "id": "call_abc", "function": { "name": "bash", "arguments": "" } }] } }

buffer[0] = {id: "call_abc", name: "bash", arguments: ""}

2
tool_calls (build args)
{ "delta": { "tool_calls": [{ "index": 0, "function": { "arguments": "{\"command\":\"ls" } }] } }

arguments += '{"command":"ls' → buffer[0].arguments = '{"command":"ls'

3
tool_calls (build args)
{ "delta": { "tool_calls": [{ "index": 0, "function": { "arguments": " -la\"}" } }] } }

arguments += ' -la"}' → buffer[0].arguments = '{"command":"ls -la"}' ✅

4
finish
{ "delta": {}, "finish_reason": "tool_calls" }

Stream done! content is empty but has tool_calls

✅ Final Result (has tool_calls but no content)
{ role: "assistant", content: null, // null! No text response reasoning_content: "", // empty tool_calls: [{ // has tool calls id: "call_abc", type: "function", function: { name: "bash", arguments: '{"command":"ls -la"}' } }] // assembled from 3 chunks }

Scenario: Everything Mixed! (Most Complex)

User: "Search for files and tell me what you find"

What streams: Reasoning → Tool Call → (wait for result) → Content

This is what you see in our AI-POC app!

1-5
Phase 1: Reasoning
{ "delta": { "role": "assistant", "reasoning_content": "The user wants" } } { "delta": { "reasoning_content": " me to search" } } { "delta": { "reasoning_content": " for files." } }

reasoning = "The user wants me to search for files."

6-10
Phase 2: Tool Call (built piece by piece)
{ "delta": { "tool_calls": [{ "index": 0, "id": "call_xyz", "function": { "name": "find", "arguments": "" } }] } } { "delta": { "tool_calls": [{ "index": 0, "function": { "arguments": "{\"path\":\".\"" } }] } } { "delta": { "tool_calls": [{ "index": 0, "function": { "arguments": ",\"pattern\":\"*\"}" } }] } }

arguments accumulated: "{\"path\":\".\",\"pattern\":\"*\"}" → Assembly complete!

11
Phase 2 done: tool_calls finish
{ "delta": {}, "finish_reason": "tool_calls" }

Round 1 done! Must execute tool and send result back

PAUSE
⏸️ Pause here! Must execute tool first

Middle steps:

1. Execute tool: find(path=".", pattern="*")

2. Got result: ["main.go", "README.md", "go.mod"]

3. Send result back in messages array with role: "tool"

12-15
Phase 3: Content (after getting result)
{ "delta": { "content": "I found" } } { "delta": { "content": " 3 files:" } } { "delta": { "content": " main.go, README.md, go.mod" } }

content = "I found 3 files: main.go, README.md, go.mod"

16
Phase 3 done: stop
{ "delta": {}, "finish_reason": "stop" }

Round 2 done! Complete answer received.

✅ Full Conversation (2 rounds)
// Round 1: User asks → AI requests tool messages = [ { role: "user", content: "Search for files..." }, { // AI responds (assembled from stream) role: "assistant", content: null, reasoning_content: "The user wants me to search...", tool_calls: [{ id: "call_xyz", function: { name: "find", arguments: "{...}" } }] }, { // Tool result (we send back) role: "tool", tool_call_id: "call_xyz", content: "[main.go, README.md, go.mod]" }, { // Round 2: AI responds after result (assembled from stream again) role: "assistant", content: "I found 3 files: main.go, README.md, go.mod", reasoning_content: "", tool_calls: [] } ]
✨ Final Answer
Answer
"The entire response must be assembled!"
Not just tool calls, but content and reasoning also accumulate from chunks

Summary of Each Assembly Type

Scenario What to assemble Difficulty
💬 Content Only content += delta.content ⭐ Easy
💭 Reasoning + Content reasoning += delta.reasoning_content then content += delta.content ⭐⭐ Medium
🔧 Tool Calls map[index] += arguments (object accumulation) ⭐⭐⭐ Hard
🌊 Mixed (Everything) All combined + handle finish_reason too ⭐⭐⭐⭐ Complex

Rule of Thumb: In Stream mode (stream: true) everything comes in small pieces! Nothing comes complete in one go (except role which comes once in the first chunk)

- Content → Concatenate strings

- Reasoning → Concatenate strings

- Tool Calls → Concatenate argument strings but use map to store objects
