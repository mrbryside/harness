🎓 API Response Explained

Understanding how streaming responses work with reasoning + content

📋 Overview

This is a streaming response from an OpenAI-compatible API (qwen3.6-plus model). The response comes back in small pieces called chunks via Server-Sent Events (SSE).

Each chunk contains a tiny piece of text. The client (your browser) receives these chunks one by one and stitches them together to show the full response.

🧩 Understanding Chunks

What is a chunk?

A chunk is one small piece of the response. In this case, there were 23 chunks total. Each chunk contains:

{
  "id": "chatcmpl-cee71616-12a7-42a0-812c-c60eb976b60a",   // Same ID for all chunks
  "object": "chat.completion.chunk",   // Always this type
  "created": 1780038321,   // Unix timestamp
  "model": "qwen3.6-plus",   // Model name
  "choices": [
    {
      "index": 0,   // Always 0 for single response
      "finish_reason": null,   // null until last chunk
      "delta": {
        // The actual content changes here!
      }
    }
  ]
}

Three Types of Chunks

Looking at the delta field, chunks come in 3 flavors:

1
🔵 reasoning_content chunks
Chunks 1-18 contain reasoning_content
The AI is thinking out loud. These are internal thoughts, not shown to user by default.
2
🟢 content chunks
Chunks 19-21 contain content
The actual response text sent to the user. "Hello! How can I help you today?"
3
🔴 finish chunk
Chunk 22 has finish_reason: "stop"
Signals the end of the stream. Empty delta means no more content.

Key insight: The model first thinks about what to do, decides no tools are needed, then plans the response.

🏁 Phase 3: Finish (Chunk 22)

The final chunk signals completion

{
  "delta": {},   // Empty! Nothing more to send
  "finish_reason": "stop"   // Model stopped naturally
}

finish_reason: "stop" means the model completed its response normally. Other possible values:

Value Meaning
"stop" Model finished naturally (this case)
"length" Hit max token limit
"tool_calls" Model wants to use a tool
null Still streaming (intermediate chunks)
🛡️ Content Filter Results

Safety checks on every chunk

Each chunk includes content filtering results (this is from Azure/OpenAI):

{
  "content_filter_results": {
    "hate": { "filtered": false },
    "jailbreak": { "detected": false, "filtered": false },
    "profanity": { "detected": false, "filtered": false },
    "self_harm": { "filtered": false },
    "sexual": { "filtered": false },
    "violence": { "filtered": false }
  }
}

All false means the content passed all safety checks. If any were true, the content would be blocked.

📝 Summary

What you learned:

1. Streaming responses come in many small chunks, not one big response

2. Reasoning happens first (hidden thoughts), then content (visible response)

3. Each chunk has a delta with tiny pieces that get stitched together

4. The finish_reason tells you why the model stopped

5. Content filters check every chunk for safety
