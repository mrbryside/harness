🔧 How Tool Call Streaming Works

การประกอบ Tool Call จาก Stream Chunks ทีละชิ้น

Key Concept
Tool Call ที่สมบูรณ์ = ประกอบจากหลาย chunks
แต่ละ chunk มีแค่เศษเสี้ยว ต้องต่อกันถึงได้ Tool Call สมบูรณ์
🎯 Overview: เกิดอะไรขึ้น?

เหตุการณ์: ผู้ใช้ถาม "List files in current directory"

AI ตัดสินใจ: จะใช้ tool bash พร้อม arguments {"command":"ls -la"}

ปัญหา: ในโหมด stream AI ไม่ได้ส่ง tool call ทั้งหมดในครั้งเดียว!

แต่ส่งมาเป็นชิ้นเล็กๆ: ทีละ field ทีละตัวอักษร ผ่านหลายๆ chunks

❓ ทำไมถึงต้องประกอบ? มาดูตัวอย่าง

สิ่งที่เราอยากได้ (Non-Stream)

แบบนี้สมบูรณ์ ใช้งานได้เลย ✓

สิ่งที่เราได้จริงๆ จาก Stream (หลาย chunks)

Chunk 1: { "tool_calls": [{ "index": 0, "id": "call_abc", "type": "function", "function": { "name": "bash", "arguments": "" } }] }

Chunk 2: { "tool_calls": [{ "index": 0, "function": { "arguments": "{\"" } }] }

Chunk 3: { "tool_calls": [{ "index": 0, "function": { "arguments": "command" } }] }

Chunk 4: { "tool_calls": [{ "index": 0, "function": { "arguments": "\":\"" } }] }

Chunk 5: { "tool_calls": [{ "index": 0, "function": { "arguments": "ls" } }] }

Chunk 6: { "tool_calls": [{ "index": 0, "function": { "arguments": " -la" } }] }

Chunk 7: { "tool_calls": [{ "index": 0, "function": { "arguments": "\"}" } }] }

ตกใจมั้ย? arguments มาทีละตัวอักษร! 😱

🔑 Key Fields ที่ใช้ประกอบ

ทำไมต้องใช้ index?

AI อาจเรียกใช้หลาย tools พร้อมกัน (parallel tool calls)

// ถ้า AI เรียก 2 tools พร้อมกัน:
{ "tool_calls": [{ "index": 0, "function": { "name": "bash", ... } }] } // tool แรก
{ "tool_calls": [{ "index": 1, "function": { "name": "find", ... } }] } // tool ที่สอง

index บอกว่า chunk นี้เป็นของ tool call ไหน

Field ใช้ทำไม?
tool_calls[index].index ระบุว่าเป็นของ tool call ลำดับที่เท่าไหร่ (0, 1, 2...) เผื่อเรียกหลาย tools พร้อมกัน
tool_calls[index].id ID ของ tool call นั้นๆ ใช้อ้างอิงตอนส่ง result กลับ
tool_calls[index].function.name ชื่อ tool ที่จะใช้ มาครั้งเดียวในชิ้นแรก
tool_calls[index].function.arguments JSON string ของ arguments สะสมจากหลาย chunks! ต้องใช้ +=
finish_reason เหตุผลที่หยุด "tool_calls" = หยุดเพราะจะใช้ tool
💻 Algorithm: โค้ดประกอบยังไง?

Pseudocode

// 1. สร้าง buffer เปล่า
toolCallBuffer = {} // map[index] = toolCall

// 2. อ่านแต่ละ chunk
for each chunk in stream:
  for each tc in chunk.delta.tool_calls:
    idx = tc.index
    
    // ถ้ายังไม่มีใน buffer สร้างใหม่
    if toolCallBuffer[idx] is empty:
      toolCallBuffer[idx] = {
        id: tc.id,
        type: tc.type,
        function: {
          name: tc.function.name,
          arguments: "" // เริ่มต้นว่าง
        }
      }
    
    // ถ้ามี arguments มา ให้ต่อทีหลัง
    if tc.function.arguments exists:
      toolCallBuffer[idx].function.arguments += tc.function.arguments

// 3. เมื่อจบ stream แปลงเป็น array
result = []
for i from 0 to len(toolCallBuffer)-1:
  result.push(toolCallBuffer[i])

สรุป: ใช้ map/dict เก็บตาม index แล้วต่อ arguments ด้วย += (string concatenation)

📝 Summary

สรุปการประกอบ Tool Call จาก Stream:

1. สร้าง map[index] → toolCall เพื่อเก็บข้อมูล

2. อ่านแต่ละ chunk แล้ว += arguments เข้า buffer

3. ใช้ index แยก tool call แต่ละอัน (เผื่อ parallel calls)

4. รอ finish_reason: "tool_calls" ถึงรู้ว่าครบ

5. แปลง buffer เป็น array แล้ว execute tools

สิ่งที่ต่างจาก Content Streaming

Content Stream Tool Call Stream
ใช้งาน แสดงผลทันที ต้องรอครบก่อน (สะสม)
การสะสม content += delta.content arguments += delta.tool_calls[n].function.arguments
จบ stream finish_reason: "stop" finish_reason: "tool_calls"
ต่อไป แสดงผลเลย ต้อง execute tool → ส่ง result → เรียก API อีกรอบ
